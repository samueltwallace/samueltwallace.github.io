Coding

I have coded in the following languages (listed in the order I remember them):
  • Java
  • Python
  • C
  • Go
  • MATLAB
  • Julia

I wouldn't call myself proficient in any particular language; I don't often stick on one language enough to know it very well (which is a flaw in my learning pattern). I code for fun, (currently) not for work or any paid ventures, or really any ventures other than my own personal use. I prefer object-oriented programming, I think it gives more structure to a program and it eases my own thinking about the organization and flow of a program.


Many professional coders refer to themselves as engineers; I do not like that term. I think coders range in expertise and skills, and do not work (generally) on public infrastructure. Essentially, the reasons for a lack of Professional Engineer's exam and its enforcement speak to my main points: there is no oversight on the projects professional coders work on. This oversight is important for ensuring engineers put out services in the public's interest. I don't consider professional coders engineers because of the lack of oversight.
